# Contributing to Prescripto

First of all, thank you for considering contributing to Prescripto.

We welcome contributions that help improve the project, fix bugs, enhance performance, improve documentation, or introduce useful features.

## Getting Started

### 1. Fork the Repository

Fork the repository to your GitHub account and clone it locally.

```bash
git clone https://github.com/your-username/prescripto.git
cd prescripto
```

### 2. Create a New Branch

Create a branch for your changes.

```bash
git checkout -b feature/your-feature-name
```

Examples:

```bash
feature/payment-refund
feature/real-time-chat
fix/appointment-bug
docs/readme-update
```

---

## Development Setup

### Install Dependencies

#### Frontend

```bash
cd frontend
npm install
```

#### Backend

```bash
cd backend
npm install
```

#### Admin

```bash
cd admin
npm install
```

---

## Coding Guidelines

Please follow these guidelines before submitting changes:

* Write clean and readable code.
* Use meaningful variable and function names.
* Keep components modular and reusable.
* Follow existing project structure.
* Remove unused code and imports.
* Test changes before submitting.

---

## Commit Message Convention

Use clear and descriptive commit messages.

Examples:

```bash
feat: add appointment refund system
fix: resolve doctor dashboard bug
docs: update README
refactor: optimize appointment controller
```

---

## Pull Request Process

1. Fork the repository.
2. Create a feature branch.
3. Commit your changes.
4. Push the branch.

```bash
git push origin feature/your-feature-name
```

5. Open a Pull Request.
6. Clearly describe the purpose of your changes.
7. Wait for review and approval.

---

## Reporting Issues

If you find a bug, please create an issue containing:

* Clear description of the problem
* Steps to reproduce
* Expected behavior
* Screenshots (if applicable)
* Environment details

---

## Feature Requests

Feature suggestions are welcome.

When creating a feature request, include:

* Problem statement
* Proposed solution
* Expected benefits
* Possible implementation approach
